1. Executive Summary

WMS Pro is a full-stack, enterprise-grade Warehouse Management System built on Next.js 14 (frontend) and Spring Boot 3 / Java 17 (backend) with a MySQL relational database. It provides end-to-end traceability of warehouse operations — from supplier goods receipt through putaway, order picking, packing, and final shipment — under a secure role-based access control model.

📦
Inbound & GRN
Receive purchase orders, create Goods Receipt Notes, track supplier deliveries.
🗂️
Smart Putaway
Auto-generate putaway tasks with bin-level barcode scan execution.
📊
Live Inventory
Real-time bin-level stock ledger with multi-state tracking and adjustments.
🛒
Order Fulfillment
Sales orders drive picking tasks, trolley assignment, packing, and shipping.
🔍
Barcode Scanning
Every operational step verified by barcode scan — picking, putaway, packing.
🛡️
Role-Based Access
Granular permission matrix per role. JWT-secured APIs. Audit trail.
📈
Dashboard & Reports
KPI widgets, bin utilization, shipment trends, live event feed.
🤖
AI Assistant
Optional AI chat for warehouse queries, stock questions, and guidance.
Business Outcome
WMS Pro eliminates manual tracking, reduces picking errors through scan verification, provides real-time inventory visibility across all bins, and gives management full operational control with role-gated access at every module.

2.3 Security Model

JWT + Role-Based Access Control (RBAC)
Every API endpoint is guarded by Spring Security. On login, the system issues a signed JWT token. Each request must carry this token. The backend checks the user's assigned role permissions before executing any operation. Permission codes like INBOUND_RECEIVE, PICKING_VIEW, ORDERS_CREATE control fine-grained access.

4.2 Goods Receipt Note (GRN)

1
Select Open Purchase Order
Operator views all open POs from the Inbound page. Selects the matching supplier PO.
2
Enter Received Quantities
Line-by-line quantities are entered (partial receipts supported). Batch/serial numbers captured if applicable.
3
GRN Generated
System creates a GRN with all received lines. Stock state is set to RECEIVED. PO status updates to PARTIALLY_RECEIVED or RECEIVED.
4
Inventory Records Created
Individual inventory records are created per SKU per batch with quantity and state RECEIVED — ready for putaway.

Manual Inventory Adjustment

Stock Correction (INVENTORY_ADJUST permission required)
Supervisors can manually adjust stock quantities with a reason code (e.g., damage, cycle count correction). All adjustments are logged in the movement log for audit purposes.

Trolley & Compartment System
Physical trolleys are registered in the system with unique barcodes. Each trolley has multiple compartments (rack slots). Before picking, a specific compartment is assigned to a sales order. This ensures picked items for different orders are kept separate on the same trolley.

10. Users, Roles & Permissions

WMS Pro uses a fully flexible Role-Based Access Control system. Admins create roles, assign granular permission codes to each role, and assign roles to users. All API endpoints enforce these permissions on every request.

Suggested Role Structure

RoleTypical PermissionsScope
Admin ALL Full system access including user management, master data, all operations
Warehouse Manager MASTER_VIEW INBOUND_VIEW INBOUND_RECEIVE ORDERS_CREATE REPORTS_VIEW DASHBOARD_VIEW All operational views + create orders + reports
Supervisor INBOUND_VIEW PUTAWAY_VIEW ORDERS_VIEW PICKING_VIEW PACKING_VIEW SHIPPING_VIEW Can view and execute all operational modules
Picker PICKING_VIEW TROLLEYS_VIEW Pick tasks and trolley compartments only
Packer PACKING_VIEW Packing station screen only
Receiver INBOUND_VIEW INBOUND_RECEIVE PUTAWAY_VIEW Inbound receiving and putaway execution

Key Business Benefits

✓
Zero Mislocation Errors
Dual-scan verification (item + bin) during putaway and picking prevents stock being placed or picked from wrong locations.
✓
Real-Time Inventory Accuracy
Every scan updates stock state and bin occupancy instantly. No overnight batch updates — managers always see live data.
✓
Faster Order Fulfillment
Auto-generated pick tasks with optimized bin paths, trolley assignment, and packing manifests reduce order cycle time.
✓
Complete Audit Trail
Every user action is tied to a user account. Movement logs track all stock transitions. Management can investigate any discrepancy.
✓
Flexible Role-Based Access
Granular permission codes mean operators only see what they need to. Sensitive functions (user management, adjustments) are admin-only.
✓
Scalable Architecture
Spring Boot REST + Next.js frontend can scale horizontally. MySQL handles large transaction volumes. New modules can be added without disrupting existing flows.
